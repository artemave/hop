# Keep the spec aligned

Keep `hop_spec.md` aligned with the implemented command contract and session behavior as the project evolves.

The spec is the canonical product description for `hop`, so implementation work should not quietly diverge from it. When a task changes user-visible behavior, command semantics, target resolution, session invariants, or integration expectations, it should either preserve the existing spec contract or update `hop_spec.md` and any derived dust facts in the same change.
